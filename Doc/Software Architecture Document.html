Software Architecture Document

1.                  Introduction

This document provides a comprehensive architectural overview of the system, using a number of different architectural views to depict different aspects of the system.  It is intended to capture and convey the significant architectural decisions which have been made on the system.

2.                  Architectural Representation

[This section describes what software architecture is for the current system, and how it is represented. Of the Use-Case, Logical, Process, Deployment, and Implementation Views, it enumerates the views that are necessary, and for each view, explains what types of model elements it contains.]

inherent MVC

3.                  Architectural Goals and Constraints

In our application we use the inherent Android MVC pattern. This means that the View is represented by Android Activity classes. Theses are the application's primary user interface components which host also other user interface components like Fragments or Views.
The Controller is mainly represented by Androids Service classes and also other classes which mediate between the view and the model or implement the application's logic. This includes for example Androids adapter classes.
