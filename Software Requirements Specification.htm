Software Requirements Specification

1.                  Introduction

1.1               Purpose

The Software Requirements Specification should give a detailed description of the Android application "If I Were You". This includes a description of the external behaviour of the app as well as a description of requirements and use-cases. Consequently it should provide the developers of this application with information how it's functions should finally look like.

1.2               Scope

This Software Requirements Specification applies to the native Android Application "If I Were You". This application is intended to enable users to challenge friends from all over the world with several tasks they will have to do. These tasks or challenges are managed in the application. Apart from that there are many other functions the users will be able to access (see 2). This document describes five Use-Cases for the application which will be finished until December 2014.

1.5               Overview

In section two of this document the idea and the functions of this application will be described in detail. Furthermore this section will also include a Use-Case diagram and a description of these use cases. Afterwards, in section three, specific requirements will be defined and clarified.

2.                  Overall Description

Functions:

  • Integration with Facebook to invite friends or to post (completed) challenges
  • Rank and see top challenges
  • Create groups to play with selected friends
  • Set challenge or group visibility (When marked as private the challenge or group is only visible by the appropriate members. When marked as public everybody who is using the app can see the challenge or group)
  • Upload a proof of a fulfilled task (e.g. a picture or video)
  • Look up proposals for possible tasks (provided either by the app itself or by browsing public groups/ challenges)

3.                  Specific Requirements

3.1               Use-Case Reports

There are three different actors namely an unregistered actor, a registered actor and a user who has already logged into the application.
The unregistered actor needs to sign for the application to be able to use it. To do so, he has two possible options: Firstly, he can register with his e-mail-address and secondly he has the possability to register with his Facebook account to be able to use the integration with Facebook of the app.
A registered actor needs to log into the application. Depending on how he has enrolled he can either log in with his e-mail-address or his Facebook account.
The user already logged into the application can use the different functionalities the application provides:
One alternative is to look up top challenges which means that he can view a list of challenges which have one of the best ratings. These ratings are done by the users which means that the user can also rate a challenge that he has looked at.
Apart from that the user can deal with his own challenges:
That means that he can view all his past challenges, fulfil or decline a challenge of a friend and create a new challenge one of his friends will have to do.
Finally there is the possibility to maintain contacts. A user can view his contacts, add a contact (who is already registered) or invite a friend to use the application.
